Перелинковка AdvOpt

Возможность перелинковки организована в соответствии с требованиями ООО "Адвантика"

Данные для перелинковки заносятся в форму добавления метаинформации.

Поле Информация для перелинковки предназначено для занесения ключевых фраз и ссылок.

Вы заносите информацию в форму построчно. Модуль определяет два типа ключевых символов: || - символ, который указывает модулю на то, что следующая строка является ключевой фразой страницы (описанием страницы); конструкция #anchor# служит для оборачивания анкоров ссылок. Прочтите, пожалуйста, инструкцию до конца, чтобы понять суть этих терминов.

Каждая страница-ловушка имеет описание и набор ссылок. Описание всегда выводится на странице-ловушке и не может являться ссылкой. Для создания описания, начните строку с символа ||.

Не используйте в описании ссылок или символов #.

Вы можете создать для одной страницы несколько описаний. Описания должны располагаться на новых строках и начинаться с символов ||.

Каждая страница может иметь набор ссылок. Эти ссылки отображаются на нецелевых страницах и ведут на страницу-ловушку.

Для того, чтобы определить новую ссылку, ведущую на страницу-ловушку, начните новую строку. В новой строке обязательно оберните слова, которые будут являться анкором ссылки в символы#

Например, для главной страницы сайта advantika.ru вся композиция описаний и ссылок будет выглядеть так:

|| Предлагаем продвижение сайтов в Хабаровске

Предлагаем #продвижение сайтов в Хабаровске#

С нашей помощью вы можете #продвинуть сайт#

#Поисковое продвижение сайтов#

|| SEO в Хабаровске

#SEO в Хабаровске#

#СЕО в Хабаровске#

#Поисковое SEO в Хабаровске#

Наша компания предлагает услуги #SEO# оптимизации

Ключевая фраза для страницы обязательно начинается с символов ||.

Данное обозначение явно указывает системе на то, что следующая фраза является ключевой для текущей страницы.

Каждая новая фраза или ссылка должна вноситься в форму с новой строки. Текст ссылок оборачивайте в символы #.

Например, строка Быстрая и #профессиональная разработка сайтов# после обработки превратится в ссылку Быстрая и профессиональная разработка сайтов

Модуль AdvOpt позволяет обрабатывать геозависимые фразы. Хорошим тоном считается отделение таких приставок как В Хабаровске или В Москве от ключевой фразы.

Для этой цели существует три текстовых поля: Приставка для перелинковки 1, Приставка для перелинковки 2 и Приставка для перелинковки 3.

Например, модернизируем наш предыдущий запрос: Быстрая и #профессиональная разработка сайтов# {%1%}. Теперь, в поле Приставка для перелинковки 1 внесём фразу в Хабаровске.

Если в блоке перелинковки не включено удаление приставок, наш результат будет таким: Быстрая и профессиональная разработка сайтов в Хабаровске

Посмотрите, как мы можем изменить приведённый выше пример:

|| Предлагаем продвижение сайтов {%1%}

Предлагаем #продвижение сайтов {%1%}#

С нашей помощью вы можете #продвинуть сайт#

#Поисковое продвижение сайтов#

|| SEO {%1%}

#SEO {%1%}#

#СЕО {%1%}#

#Поисковое SEO {%1%}#

Наша компания предлагает услуги #SEO# оптимизации

Отображение перелинковки

Перелинковка выводится на сайте блоком. Перейдите в меню управления блоками, найдите блок Перелинковка и разместите его в любом удобном для вас месте.

В настройках блока имеется возможность тонкой настройки отображения.

Заголовок блока - это стандартное поле. Оно позволяет выводить наш блок под заголовком. В большинстве случаев, вы должны ввести в поле значение , чтобы не выводить заголовок вовсе.

Отображение footer - это возможность определить отображение описания на странице

  • Первое вхождение - идеально подходит к случаям, когда к странице привязано одно описание. В этом режиме модуль выводит именно это описание на странице-ловушке
  • Случайное вхождение - если вы создали для странице несколько описаний (строк, которые начинались с ||), с помощью этого режима вы будете выводить на странице-ловушке случайное описание из всего списка описаний.
  • Все вхождения - если вы создали для странице несколько описаний (строк, которые начинались с ||), с помощью этого режима вы сможете вывести все эти описания через запятую

Удалять приставки в footer - если опция включена, приставки для перелинковки ({%1%}, {%2%}, {%3%}) в описаниях страниц не будут заменены, а будут удалены

Отображение ссылок - аналог Отображения footer, но только он применяется для ссылок, отображаемых на нецелевых страницах.

  • Привязанная ссылка - привязывает к нецелевой странице только одну ссылку из всего массива. Ссылка может измениться, только если вы добавите ссылку в любую страницу или удалите любую ссылку.
  • Случайная ссылка - на нецелевых страницах будет выводиться случайная ссылка из всего массива.
  • Список привязанных ссылок - выводит список из нескольких привязанных ссылок
  • Список случайных ссылок - выводит список из нескольких случайных ссылок
  • Облако привязанных тегов - выводит список из нескольких привязанных ссылок и присваивает ему класс landing_ul_cloud_relative, который позволяет стилизовать список в облако тегов.
  • Облако случайных тегов - выводит список из нескольких случайных ссылок и присваивает ему класс landing_ul_cloud_relative, который позволяет стилизовать список в облако тегов.

Удалять приставки в ссылках - если опция включена, приставки для перелинковки ({%1%}, {%2%}, {%3%}) в описаниях страниц не будут заменены, а будут удалены

Заголовок для footer - позволяет вывести перед описанием страницы в нашем блоке заголовок. Например,

Вы могли найти нашу страницу по запросам:
...

Заголовок для ссылок - позволяет вывести перед ссылками в нашем блоке заголовок. Например,

Смотрите на нашем сайте:
...

Когда вы выводите блок на сайте, помните, что он отображается точно так же, как и все остальные блоки.

В многочисленных случаях, вам может потребоваться открыть файл .css сайта и стилизовать блок для корректного отображения

Блок появляется автоматически в панели управления сайта -> Структура -> Блоки

Вы можете найти блок с именем "Перелинковка" в списке блоков, присвоить ему регион, сохранить, и блок будет отображён на сайте. Если этого не произошло, очистите кэш. Также, можно поискать блок в исходном коде страницы.
